Survey of Assistive Vision Systems

⏱️ 1.5 minutes

In the field of assistive technology, computer vision (CV) has historically been applied to broader accessibility challenges like indoor/outdoor navigation, reading signs, and object recognition for the visually impaired. However, the recent shift toward fine-grained object-finding - the task of helping users locate their own, specific items in real-world settings - marks a significant evolution in this space.

Early work such as [1] provides a comprehensive overview of CV-based assistive technologies, highlighting systems focused on navigation, gesture recognition, and basic scene understanding. These systems often rely on either wearable cameras or smartphone input to deliver contextual cues to users. However, they rarely support custom item recognition or real-time object tracking.

The work by [2] introduced ObjectFinder, a smartphone-based system that uses open-vocabulary detection to allow blind users to find everyday objects by name. It uses CLIP-like embeddings to match a target word to an object in the visual scene. The system was evaluated through real-world use by blind participants, revealing high usability, though failure cases increased in cluttered spaces or with ambiguous labels.

Similarly, [3] proposed OpenGuide, which integrates vision-language detection with physical guidance, using AR navigation and scene graphs to help users find items in indoor environments. While OpenGuide uses heavier computation and sensor fusion, it shows strong potential for multi-step retrieval and spatial understanding - something earlier systems lacked.

A more performance-optimized approach is FUS3D by [4], which combines detection and tracking into a unified 3D system with lower runtime cost. Though not specifically built for assistive scenarios, its design aligns well with real-time, on-device object-finding in homes or care facilities.

Overall, these systems form the foundation for assistive object-finding solutions, but vary significantly in target users, device requirements, vocabulary flexibility, and interface design. A critical takeaway from this survey is that no single solution fits all use cases. Researchers must consider hardware limitations, user cognition, guidance modality, and personalization to create robust, accessible systems.
